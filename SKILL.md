# Talking head edit

## Two roles

| Role | Tool | Job |
|------|------|-----|
| **Director** | `scripts/director.py` | Plan delivery, chapters, exclusions; get user OK |
| **Editor** | `scripts/batch.py`, `stitch_chapters.py` | Transcribe, cut, caption, render |

**You are the director first.** Do not batch-render or stitch until the user confirms the plan.

## When to use

- Stage speech, interview, course talking-head, conference clips
- Many short takes from one session
- User wants **tighter pace**, not a restructured story

## Director pipeline (required)

```
plan → discuss → decisions.json → sample → caption OK → batch → [stitch]
```

### Step 1 — Plan (before any render)

```bash
python scripts/pipeline.py plan
```

Read `edit/director/brief.md`. Present to user:

- Clip inventory table (duration, flags, preview text)
- Delivery options: **individual** | **chapters** | **full_stitch** | **highlights**
- Recommended option + proposed chapter groupings
- Clips to exclude (fragments, false "thank you" takes)
- **Ask explicit questions** — do not assume

See [DIRECTOR.md](./DIRECTOR.md) for the full playbook.

### Step 2 — Record decisions

Edit `edit/director/decisions.json` after user confirms:

- `delivery` — chosen format
- `exclude_stems` — fragments to skip in stitch
- `stitch.chapters` — if delivery is chapters
- `sample.stem` — clip for caption review

```bash
python scripts/director.py apply    # sync config/chapters.json
python scripts/director.py validate
```

### Step 3 — Execute (editor)

```bash
python scripts/pipeline.py sample IMG_XXXX
# user approves captions → set sample.caption_approved + batch.approved in decisions.json
python scripts/pipeline.py batch
# if chapters → set stitch.approved → 
python scripts/pipeline.py stitch
```

## Hard rules

1. **Director before editor** — never batch/stitch without plan + user confirmation
2. **Show the plan in plain English** — like offering Option A/B/C/D, not silent execution
3. **Sample before batch** — caption approval is separate from delivery approval
4. **Pacing cleanup only** — chronological order; cut pauses/filler; no thematic restructure
5. **Subtitles last** on chapter stitches; stage `manrope-speech` or social ASS (see WORKFLOW §11)
6. **Transcript → edit** — when the user edits markdown: DELETE → EDL cuts, FIX → caption text only (WORKFLOW §12)
7. **Self-eval before handoff** — pull `verify/` frames at cuts/bookends/caption risks; max 3 fix loops (WORKFLOW §13)

## Delivery options (always offer)

| Option | When |
|--------|------|
| individual | Social / modular — no stitch |
| chapters | 3–4 topic videos (~1–3 min) — **default for 20+ clips** |
| full_stitch | Only if user explicitly wants one long file |
| highlights | ~90s promo reel |

## Config

- `config/defaults.json` — pause threshold, subtitle style
- `edit/director/decisions.json` — user-approved plan (gates batch/stitch)
- `config/chapters.json` — generated by `director.py apply`

## Demo + talking-head PiP

When the deliverable is **screen demo + corner talking head** (CapCut-style rounded rect):

```bash
python helpers/compose_pip.py still --preset capcut_0716 \
  --demo DEMO.mov --head HEAD.mov --t 5 -o edit/verify/pip_5s.png
python helpers/compose_pip.py render --preset capcut_0716 \
  --demo DEMO.mov --head HEAD.mov --audio head -o edit/preview_pip.mp4
```

Preset `capcut_0716` is reverse-engineered from CapCut draft `0716` (material face-crop → scale 0.32 → rounded mask → bottom-right). Params live in `config/defaults.json` → `pip` and `helpers/compose_pip.py`. Confirm a still against the user's CapCut reference before a long render.

## Social cut extras

For vertical reels: ASS burn @1× → content 1.2× → 2.8s title card; defaults in `config/defaults.json` (`caption_ass_social`, `title_card`).
